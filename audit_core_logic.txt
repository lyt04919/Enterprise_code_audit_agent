【项目申报参考材料】

项目解决的核心痛点：
1. 高级研发资源错配：资深工程师在基础代码 Review 上耗费巨大，形成研发瓶颈。
2. 代码生成质量与安全隐患：单一模型生成的代码常含有隐蔽漏洞，缺乏多维度的对抗式检查。
3. 长文本上下文断层：AI 难以理解跨文件的深层业务逻辑，导致重构风险高。

核心逻辑流：
构建了“全景解析-多维审查-攻防对抗-反思修正”的四阶段多 Agent 协作流：
- 架构师 Agent (Architect): 解析代码变更的全局业务意图。
- 审计 Agent (Reviewer): 利用长链推理定位逻辑隐患与规范问题。
- 安全 Agent (Security): 模拟攻击路径，针对性挖掘越权或注入漏洞。
- 反思修正流 (Reflection Loop): 汇总专家组意见，由开发 Agent 进行代码重构，并通过多次内部迭代确保最终输出的置信度。
